# PJKR — 個人用プロジェクト管理支援ツール

システム部門の個人業務(タスク・ボール管理・報告)を支援するローカルツール。
詳細は [docs/要件定義書.md](docs/要件定義書.md) / [docs/設計書.md](docs/設計書.md) を参照。

## 使い方

```powershell
go build -o pjkr.exe .   # ビルド(初回のみ)
.\pjkr.exe               # 起動 → ブラウザが自動で開き、タスクトレイに常駐
```

- ダブルクリックでも起動できます。ブラウザを閉じても常駐は続き、**終了はタスクトレイの「PJKR」アイコン右クリック→終了**です
- コンソールを出したくない場合は `go build -ldflags "-H windowsgui" -o pjkr.exe .` でビルドしてください
- データは exe と同じ場所の `data/pjkr.json` に自動保存されます(バックアップは `data/backup/` に20世代)
- 毎朝の通知・Windows起動時の自動起動は設定画面からON/OFFできます
- 会社PCで exe が実行できない場合: `go run .` で起動してください

### オプション

| フラグ | 説明 |
|---|---|
| `-port 8760` | 待ち受けポート(使用中なら自動で+19まで探索) |
| `-data <dir>` | データフォルダの場所を変更 |
| `-nobrowser` | 起動時にブラウザを開かない |
| `-notray` | タスクトレイに常駐しない(開発・検証用。Ctrl+Cで終了) |

### ショートカットキー

| キー | 動作 |
|---|---|
| `Ctrl+I` | Inboxに書き留め(どの画面からでも) |
| `Ctrl+F` | 横断検索(タスク・課題・メモ・Inbox) |
| `Ctrl+Z` / `Ctrl+Y` | 元に戻す / やり直し |
| `Esc` | パネル・ダイアログを閉じる |

## 主な使い方(ボール管理)

1. 依頼を出したらタスクを **「返答待ち」** に切り替え(相手・待ち開始日を記録)
2. 5営業日(設定可)返答がないと **「催促候補」** としてダッシュボード・ボール画面に浮上
3. 質問を受けたら Inbox → **「判断保留」** へ(誰から・期限・確認メモを記録)
4. 回答したら **「回答して完了」** で概要を記録 → **回答管理** 画面からいつでも検索可能

## 報告(週報)

1. **報告** 画面で週を選んで「下書きを生成」— 完了タスク・作業ログ・課題・ボール状況から案件別の下書きが自動生成される
2. 画面上で文面を調整して「保存」(過去の週報も参照可能)
3. 出力は3形態: **報告資料モード**(そのまま見せる・印刷)/ **Copilot用にコピー**(貼り付けて清書)/ **Excel出力**

## ガントチャート

- **ガント** 画面でタスクの期間(開始日〜期限)とマイルストーン(🚩 リリース日・レビュー日など)を時系列表示。週/月/四半期の切替、赤線が今日
- バーをドラッグで日程移動、両端ドラッグで期間変更(Ctrl+Zで取消可)。クリックでタスク詳細へ
- 開始日・期限のどちらも未設定のタスクは表示されません(タスク詳細で設定してください)

## データ入出力

- **データ入出力** 画面から、タスク・課題・ボール・Q&AのExcelエクスポート、テンプレートによるExcel一括インポート(Ctrl+Zで取消可)、JSONバックアップ/リストアができます

## 開発

- Go 1.22+。外部依存は systray(トレイ常駐)と go-toast(通知)のみ。フロントは `web/` の素のJS(ES Modules)で、ビルドツール不要
- `web/` を編集したら `go build` で exe に再同梱される(開発中は `go run . -notray` が楽)
- テスト: `go test ./...`(営業日計算)
- 外部依存: systray(トレイ)・go-toast(通知)・excelize(Excel入出力)
- **Phase 4 まで全フェーズ実装済み**(基盤・Inbox・タスク・案件ホーム・今日ビュー・双方向ボール管理・判断保留箱・回答管理・課題リスク・横断検索・一括登録・通知・トレイ常駐・報告(週報/Copilotコピー/Excel)・Excel入出力・JSONリストア・ガントチャート・マイルストーン)。フェーズ計画は要件定義書§9参照
